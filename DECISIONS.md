# Decision log

## 2026-07-06 — Evidence-bearing pipeline instead of immediate fine-tuning

The seven-day system will use retrieval, structured extraction, comparison, and calibration. Fine-tuning is deferred until reviewed correction data exists.

## 2026-07-06 — Historically scoped cosmology cards

Reference cards represent identifiable schools, periods, or lineages. Broad religions are navigation categories, not single doctrinal vectors.

## 2026-07-06 — Three kinds of uncertainty

Evidence coverage, candidate fit, and calibrated confidence are separate. Overall confidence cannot be copied from an LLM self-report.

## 2026-07-06 — Conservative MAI integration

MAI value-card matches may be reported from available data. Moral-graph position is reported only when a real edge with compatible context and provenance exists.

## 2026-07-06 — One active task and bounded checking

The canonical state permits one active task. Two consecutive check-only cycles force implementation or an evidence-backed blocker.

## 2026-07-06 — Freeze cosmological primitive ontology version 1

The prototype will compare text and reference cards through twelve stable primitive IDs defined in `docs/COSMOLOGY_PRIMITIVES.md`. All primitives use the same four text-evidence states and require voice-aware exact spans when observed. Cosmological description remains separate from value matching, and the freeze is an implementation contract rather than a claim of universal or human-reviewed validity.
